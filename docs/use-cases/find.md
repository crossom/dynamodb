# Use Cases - Find

## Simple

- Find by primary key (not composite, the DB hasn't a sortKey)
- Find by primary key without sortKey (composite, DB has a sortKe, but the user not passed it)
- Find by index
- Find by random columns

## With find operators

- Find by primary key (not composite, the DB hasn't a sortKey)
- Find by primary key without sortKey (composite, DB has a sortKe, but the user not passed it)
- Find by index
- Find by random columns
